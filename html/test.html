Commands

Commands specific to this game or important to highlight:

  • UNDO: undo last move
  • SAVE: save game
  • RESTORE: restore saved game
  • HELP: read help about game>
  • HINTS: get context-aware hints
  • MAP: show map of area you're in
  • OBJECTS: list objects you've handled
  • WHO: list monks in room
  • WAIT x HOURS|MINUTES
  • LOOK UP topic IN book
  • LOOK UNDER thing
  • HIDE BEHIND thing
  • KNEEL ON thing
  • PRAY
  • SING

This is a game with subtle descriptions. Read carefully and try other senses, like taste, smell, and hearing.

How to Test

Turn on Transcript

If you're playing with a non-brower real interpreter: Immediately, turn on transcripting with Script On — this will record everything you type and everything the game shows. After you're done, email the transcript to joel@joelburton.com.

Try Everything

Try lots of things! Even if you think they're not helpful or a bit silly, I want to make sure the world seems real. For example, if you try this:

  > EXAMINE BODY
  You feel nothing unexpected.

It'd be much better if there was a custom message about touching a body, even if it isn't important to the plot.

Once the game is tested well, I'll add a time limit to complete your investigations by Compline. I'm not doing that now so testers won't be frustrated by a time limit.

Adding Bugtest Comments

(These only work in a non-browser real interpreter)

Anytime you catch a problem — confusing text, typos, grammar, feeling stuck — enter a comment. For example:

> X REMIGIO
Remigio is is a monk...

That's a typo! In the game, enter a line starting with an asterisk.

> * typo: is is

If you find something that feels more like a programming bug, do the same but with an exclamation mark instead: 

> X DOOR
The door is solid oak and closed.

> OPEN DOOR
The door is already open.

! Door says both open and closed
... lots of debugging info output ...

The exclamation command adds enough debugging info that it can help me fix the bug.
