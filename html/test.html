Testing Our Lady of Thorns

If you've never played interactive fiction before, this might help: IF Postcard

Commands specific to this game or important to highlight:

  • MAP: see a map of the area you're in
  • UNDO: undo last move
  • SAVE: save game state
  • RESTORE: load a saved game

Debugging commands:

These are cheating, but as you test, you might find it useful to examine or manipulate things in special ways. The real game will disable these commands.

  • GOTO room-name: jump to a room by name
  • GONEAR obj: jump to a room by an object in it
  • PURLOIN obj: steal an object from anywhere, even if its something you couldn't normally take
  • SCOPE: list all of the things in "scope" (nearby)
  • ROOMS: list all of the rooms in game
  • SETTIME h m: Set the time (eg SETTIME 14 15 sets to 2:15pm)
  • TIME: show current time

How to Test

Get the Game

Download: Thorns.z5.

Turn on transcript

Immediately, turn on transcripting with SCRIPT ON --- this will record everything you type and that it shows. After you're done, email that to me at joel@joelburton.com.

Try Everything

Try lots of things! Even if you think they're not helpful or a bit silly, I want to make sure the world seems real. For example, if you try this: 

  > EXAMINE BODY
  You feel nothing unexpected.

It'd be much better if there was a custom message about touching a body, even if it isn't important.

Also note: THERE IS NO WAY TO SECRETLY LOSE THIS GAME. Some games have challenges where if you fail to do a certain thing early in the game, or you eat candy that would be needed to bribe a dwarf later, you'll be locked out of success without even knowing it.

In this game, you can lose only in ways that are obvious when they happen: getting very ill, being caught doing something against the prior's rules, etc. So: if you feel like "OH NO, I SHOULDN'T HAVE DONE THAT", if it doesn't kill you in a few turns, it won't.

Once the game is tested well, I'll add a time limit that you have to complete your investigations by sunset. I'm not doing that now so that testers who try zillions of things won't be frustrated by a time limit.

Adding bugtesting comments

Anytime you catch any problem (confusing text, typos, grammar, feeling stuck, etc, please enter a command like this: 

  > X DOOR
  The door is solid oak and closed.

  > OPEN DOOR
  The door is already open

  > * Door reports being both open and closed

A line like that, starting with an asterisk, is ignored by the game, but it'll appear in the transcript, and I'll search especially for those.
